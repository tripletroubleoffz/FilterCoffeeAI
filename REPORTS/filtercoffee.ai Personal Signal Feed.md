# Personal Signal Feed – Phase 1 (B2C Cash Engine)
**Business + Minimal Tech Document**

---

## 1. Executive Summary

**Personal Signal Feed** is a B2C subscription product that delivers *only what changed* on topics a user cares about, along with *why it matters*.  
It is designed to generate **immediate recurring revenue** while laying the foundation for future B2B expansion.

The product solves one core problem:

> People don’t lack information.  
> They lack clarity, relevance, and confidence.

Phase 1 focuses on **email-first delivery**, **simple pricing**, and **strict cost control**.

---

## 2. Product Vision (Phase 1)

### One-line value proposition
> *A daily signal that tells you what actually changed — and why it matters to you.*

### What this is
- A **personal intelligence feed**
- Calm, concise, non-addictive
- Optimized for professionals, not news junkies

### What this is NOT
- A generic news aggregator
- A social feed
- A trading or advisory product

---

## 3. Target Customer (ICP)

### Primary ICP (Phase 1)
- Tech professionals
- Founders / indie hackers
- Product managers
- Startup / AI ecosystem followers

### Characteristics
- Reads too much news
- Feels overwhelmed
- Will pay for clarity and time savings
- Comfortable with subscriptions

---

## 4. Core User Jobs-to-be-Done

| Job | Description |
|---|---|
| Stay informed | Know what changed without scanning dozens of sources |
| Reduce anxiety | Avoid missing important updates |
| Save time | Spend <5 minutes per day |
| Build confidence | Understand *why* something matters |

---

## 5. MVP Feature Scope (Strict)

### Included (Phase 1)
- User signup/login
- Topic creation (free-text + include/exclude keywords)
- Topic subscription (daily or weekly)
- Email digest delivery
- Digest history in web app
- Stripe-based subscription
- Cost/token monitoring (internal)

### Explicitly Excluded
- Mobile apps
- Social features
- Real-time breaking alerts
- Slack / Teams delivery
- Team sharing
- Advanced personalization ML

---

## 6. Core User Flow

1. User signs up
2. User creates 1–5 topics
3. User chooses frequency (daily/weekly)
4. System sends one clean digest
5. User optionally upgrades for more topics/frequency

---

## 7. Digest Format (Product Core)

Each digest contains:

### What Changed
- 2–4 concise bullets
- Only new information since last digest

### Why This Matters
- 1–2 interpretive bullets
- Framed for professionals

### Sources
- Transparent links
- Credibility over volume

## 8. Monetization Strategy

### Pricing (example)
| Plan | Price | Limits |
|---|---|---|

| Pro | ₹499 / month | 5 topics, daily |

- Clear upgrade trigger
- Strong perceived ROI
